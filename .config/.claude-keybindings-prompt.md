## IMPORTANT: READ THIS WHENEVER YOU VIEW OR MODIFY CONFIG FILES

This file contains critical instructions for maintaining the keybindings documentation.

---

## Master Instruction

**WHENEVER you read, view, edit, or work with ANY of the following configuration files, you MUST automatically check and update the keybindings documentation:**

### Monitored Config Files

1. **Karabiner:**
   - `~/.config/karabiner/karabiner.json`

2. **AeroSpace:**
   - `~/.config/aerospace/aerospace.toml`

3. **Neovim:**
   - `~/.config/nvim/lua/config/keymaps.lua`
   - `~/.config/nvim/lua/plugins/telescope.lua`
   - `~/.config/nvim/lua/plugins/neo-tree.lua`
   - `~/.config/nvim/lua/plugins/lsp.lua`
   - `~/.config/nvim/lua/plugins/bufferline.lua`
   - `~/.config/nvim/lua/plugins/completion.lua`
   - `~/.config/nvim/lua/plugins/ui.lua`
   - Any other new plugin files with keybindings

4. **Lazygit:**
   - `~/.config/lazygit/config.yml`

5. **Any new config files** added to `~/.config/` that contain keybindings

---

## Required Actions

When you view or edit ANY of the above files:

1. **Scan for keybindings** - Look for:
   - `vim.keymap.set()` calls (Neovim)
   - `binding` sections (AeroSpace)
   - `simple_modifications` or `complex_modifications` (Karabiner)
   - Any keyboard shortcut configurations

2. **Compare with README.md:**
   - Check if the keybinding exists in `~/README.md`
   - Verify the description is accurate
   - Look for any removed keybindings

3. **Update README.md** if you find:
   - New keybindings not documented
   - Changed keybindings with outdated descriptions
   - Removed keybindings that should be deleted from docs
   - Reorganization needs

4. **Update the "Last Updated" date** at the bottom of README.md

---

## Update Format

When updating KEYBINDINGS.md:

- **Maintain the table format** for consistency
- **Group related keybindings** together
- **Use clear, concise descriptions**
- **Include mode information** for Neovim keybindings (Normal, Visual, etc.)
- **Note special modifiers** (Shift, Ctrl, Alt, Leader, etc.)
- **Keep sections alphabetically organized** where it makes sense

---

## Proactive Behavior

- **Do NOT ask permission** to update the keybindings documentation
- **Automatically update** when you detect changes
- **Inform the user** after updating with a brief summary of changes
- **Be thorough** - check all related files even if only one was modified
